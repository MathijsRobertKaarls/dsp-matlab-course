2.3 — DFT vs FFT

Module 2 · Frequency Domain · ~20 min

Introduction

We've been using fft() in MATLAB without worrying about what's happening inside. Now let's open the hood. The Discrete Fourier Transform (DFT) is the formula that computes the spectrum of a finite digital signal. The Fast Fourier Transform (FFT) is a clever algorithm that computes the exact same result, but dramatically faster.

The DFT on $N = 1{,}000{,}000$ samples takes about $10^{12}$ operations naively. The FFT takes about $20{,}000{,}000$ — 50,000 times fewer. That difference is why real-time spectrum analyzers, audio processing, and communications are possible.

Core Concept

The DFT

Given $N$ samples $x[0], x[1], \ldots, x[N-1]$, the DFT computes $N$ complex spectral values:

$$X[k] = \sum_{n=0}^{N-1} x[n] \cdot e^{-j2\pi kn/N}, \quad k = 0, 1, \ldots, N-1$$

Each $X[k]$ is the component at frequency:

$$f_k = k \cdot \frac{f_s}{N}$$

So the frequency resolution (distance between adjacent bins) is:

$$\Delta f = \frac{f_s}{N}$$
Key insight: More samples $N$ → finer frequency resolution $\Delta f = f_s/N$. But it doesn't change the highest representable frequency (that's still $f_s/2$). Think of it as a ruler: more sample = more tick marks between 0 and $f_s/2$.

The FFT — Cooley-Tukey Algorithm

The FFT exploits the fact that the DFT computation has a lot of redundancy. By recursively splitting the $N$-point DFT into two $N/2$-point DFTs (the "divide and conquer" approach), it reduces complexity from:

$$\mathcal{O}(N^2) \quad\to\quad \mathcal{O}(N \log_2 N)$$

This works best when $N$ is a power of 2 (e.g., 256, 512, 1024, 2048...). MATLAB's fft() automatically uses the FFT algorithm.

DFT Properties You Must Know

  • Linearity: $\text{DFT}\{ax + by\} = a\,X[k] + b\,Y[k]$
  • Circular shift: Time shift → phase rotation in frequency domain
  • Convolution theorem: Time convolution ↔ frequency multiplication
  • Parseval's: $\sum|x[n]|^2 = \frac{1}{N}\sum|X[k]|^2$

Interpreting FFT Output

MATLAB's fft(x) returns $N$ complex values:

  • $X[0]$: DC component (mean × N)
  • $X[1]$ to $X[N/2-1]$: positive frequencies ($f_s/N$ to $f_s/2 - f_s/N$)
  • $X[N/2]$: the Nyquist component
  • $X[N/2+1]$ to $X[N-1]$: negative frequencies (mirror of positive for real signals)

Use fftshift() to center the spectrum at DC (useful for visualizing two-sided spectra).

Mathematical Foundation

DFT as a matrix operation

The DFT can be written as a matrix multiply $\mathbf{X} = \mathbf{W} \mathbf{x}$, where the $N \times N$ DFT matrix $\mathbf{W}$ has entries:

$$W_{kn} = e^{-j2\pi kn/N} = W_N^{kn}, \quad W_N = e^{-j2\pi/N}$$

This naive multiply is $\mathcal{O}(N^2)$. The FFT achieves $\mathcal{O}(N \log_2 N)$ by exploiting the periodicity and symmetry of $W_N^{kn}$.

Frequency axis construction

For a signal of $N$ samples at rate $f_s$:

  • One-sided frequency axis (0 to $f_s/2$): f = (0:N/2-1) * fs/N
  • Two-sided (centered): f = (-N/2:N/2-1) * fs/N (use with fftshift)

Conceptual Questions

Q1. A signal is sampled at $f_s = 1000$ Hz. You compute the DFT with $N = 500$ samples. What is the frequency resolution?

  • 1 Hz
  • 2 Hz
  • 0.5 Hz
  • 500 Hz

Q2. How does the FFT differ from the DFT?

  • The FFT gives an approximate result while the DFT is exact
  • The FFT only works for audio signals
  • The FFT computes the same result as the DFT, but in O(N log N) instead of O(N²) operations
  • The FFT uses fewer samples than the DFT

Q3. What does fftshift() do to an FFT output?

  • Removes the DC component
  • Applies a Hamming window
  • Rearranges the output so DC is centered, with negative frequencies on the left
  • Doubles the frequency resolution

MATLAB Exercise

Compare DFT (manual loop) vs FFT timing, and explore frequency resolution.

%% 2.3 DFT vs FFT

%% Part A — Manual DFT vs fft() (check they agree)
N = 64;
x = randn(1, N);   % Random test signal

% Manual DFT
X_dft = zeros(1, N);
for k = 0:N-1
  for n = 0:N-1
    X_dft(k+1) = X_dft(k+1) + x(n+1) * exp(-1j*2*pi*k*n/N);
  end
end

X_fft = fft(x);
fprintf('Max DFT vs FFT difference: %.2e\n', max(abs(X_dft - X_fft)));

%% Part B — Timing: O(N²) vs O(N log N)
sizes = [64, 128, 256, 512, 1024, 2048, 4096];
t_dft = zeros(size(sizes));
t_fft = zeros(size(sizes));

for i = 1:length(sizes)
  N = sizes(i);
  x_test = randn(1, N);

  % Manual DFT timing (for moderate N only)
  if N <= 512
    tic;
    X_m = zeros(1,N);
    n_vec = 0:N-1;
    for k = 0:N-1
      X_m(k+1) = sum(x_test .* exp(-1j*2*pi*k*n_vec/N));
    end
    t_dft(i) = toc;
  else
    t_dft(i) = NaN;   % Too slow to bother
  end

  % FFT timing
  tic;
  for rep = 1:100; fft(x_test); end
  t_fft(i) = toc / 100;
end

figure('Name','DFT vs FFT Timing');
semilogy(sizes, t_dft, 'r-o', 'DisplayName', 'Manual DFT O(N²)'); hold on;
semilogy(sizes, t_fft, 'b-o', 'DisplayName', 'FFT O(N log N)');
xlabel('N (signal length)'); ylabel('Time (s)');
title('Computation Time: DFT vs FFT'); legend; grid on;

%% Part C — Frequency resolution
fs = 1000;
f0 = 100;

for N = [50, 200, 1000]
  t = (0:N-1)/fs;
  x = sin(2*pi*f0*t);
  X = fft(x);
  f_ax = (0:N/2-1)*fs/N;
  figure('Name', sprintf('N=%d',N));
  stem(f_ax, abs(X(1:N/2))*2/N, 'filled', 'MarkerSize',3);
  title(sprintf('N=%d, Δf=%.1f Hz', N, fs/N));
  xlabel('Hz'); ylabel('|X[k]|'); xlim([0 300]);
  xline(f0,'r--','100 Hz');
end
💡 Hint — Timing results

The DFT vs FFT time difference becomes dramatic at larger N. At N=512, FFT is likely 10-50× faster than the manual loop. The timing plot on a log scale should show the DFT growing much steeper (parabolic) than the FFT (nearly linear). The "max difference" in Part A should be on the order of 1e-12 — numerical noise, not a real error.

Key Takeaways

  • The DFT computes $N$ spectral values from $N$ samples: $X[k] = \sum x[n] e^{-j2\pi kn/N}$.
  • Frequency resolution: $\Delta f = f_s/N$ — more samples = finer resolution.
  • The FFT computes the same result in $\mathcal{O}(N \log N)$ instead of $\mathcal{O}(N^2)$.
  • MATLAB's fft() uses FFT automatically; works best with $N = 2^k$.
  • Use fftshift() to center the spectrum at DC for two-sided display.
