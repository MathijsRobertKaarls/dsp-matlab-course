6.3 — Downsampling (Decimation)

Module 6 · Resampling · ~15 min

Introduction

Downsampling reduces the sampling rate — you keep only every $M$th sample. This is useful for reducing data size, matching system input rates, or implementing efficient multirate filterbanks. The danger: if you just throw away samples without care, you'll alias any signal content above the new Nyquist limit. The solution is always to lowpass filter before downsampling.

Core Concept

The Decimation Process

Downsampling (decimation) by integer factor $M$:

  1. Anti-aliasing lowpass filter: cutoff at $f_s / (2M)$ — remove everything that would alias after downsampling.
  2. Downsample: keep every $M$th sample, discard the rest.
$$y[n] = x_{filtered}[nM]$$
Order matters! Always filter BEFORE downsampling. The opposite order causes aliasing that cannot be undone.

Spectral Effect of Downsampling

After lowpass filtering and downsampling by $M$, the new sampling rate is $f_s/M$, so the new Nyquist is $f_s/(2M)$. Any content that was above $f_s/(2M)$ before filtering has been removed. The spectrum of $y[n]$ is the filtered spectrum of $x[n]$, scaled in frequency by $M$.

Mathematical Foundation

The DTFT of the downsampled signal:

$$Y(e^{j\hat\omega}) = \frac{1}{M} \sum_{k=0}^{M-1} X_{LP}\!\left(e^{j(\hat\omega - 2\pi k)/M}\right)$$

This shows $M$ aliased terms — exactly like the STFT aliasing formula. The anti-aliasing LPF ensures $X_{LP}(e^{j\hat\omega}) = 0$ for $|\hat\omega| > \pi/M$, so all but the $k=0$ term vanish.

MATLAB Exercise

%% 6.3 Downsampling / Decimation

fs = 8000;
M  = 4;        % Downsample factor → new rate = 2000 Hz
T  = 0.5;
t  = (0:1/fs:T-1/fs);

% Signal with content at 200 Hz and 1800 Hz
f1 = 200;   f2 = 1800;   % f2 is above new Nyquist (1000 Hz)!
x = sin(2*pi*f1*t) + 0.8*sin(2*pi*f2*t);

%% Wrong: downsample without filtering (aliasing!)
x_wrong = downsample(x, M);
fs_new  = fs / M;

%% Correct: anti-alias filter first
Fn_aa = (fs_new/2) / (fs/2);   % normalized cutoff = 1/M
b_aa  = fir1(64, Fn_aa);
x_filtered = filter(b_aa, 1, x);
x_correct  = downsample(x_filtered, M);

%% Alternatively: use decimate() which does both steps
x_decimate = decimate(x, M);

%% Spectrum comparison
t_new  = (0:length(x_wrong)-1)/fs_new;
figure('Name','Decimation Comparison');

subplot(3,2,1); plot(t(1:400), x(1:400));
title(sprintf('Original: fs=%d Hz', fs)); xlabel('Time (s)');

subplot(3,2,2);
N = length(x);
plot((0:N/2-1)*fs/N, abs(fft(x))*2/N);
title('Original spectrum'); xlabel('Hz'); xlim([0 fs/2]);
xline(fs_new/2,'r--','new Nyquist');

subplot(3,2,3); plot(t_new, x_wrong);
title('Decimated WITHOUT anti-alias (aliased!)'); xlabel('Time (s)');

subplot(3,2,4);
Nw = length(x_wrong);
plot((0:Nw/2-1)*fs_new/Nw, abs(fft(x_wrong))*2/Nw);
title(sprintf('Aliased spectrum — 1800Hz appears at %dHz', fs_new-f2));
xlabel('Hz'); xlim([0 fs_new/2]);

subplot(3,2,5); plot(t_new, x_correct);
title('Decimated WITH anti-alias filter'); xlabel('Time (s)');

subplot(3,2,6);
Nc = length(x_correct);
plot((0:Nc/2-1)*fs_new/Nc, abs(fft(x_correct))*2/Nc);
title('Clean spectrum — only 200Hz remains'); xlabel('Hz'); xlim([0 fs_new/2]);

fprintf('\nWith anti-alias filter, 1800Hz component is removed.\n');
fprintf('Without it, 1800Hz aliases to %.0f Hz.\n', abs(f2 - fs_new));
💡 Hint

The key insight: 1800 Hz is above the new Nyquist (1000 Hz), so it aliases to |1800-2000| = 200 Hz. Without the anti-alias filter, both the true 200 Hz and the aliased 1800 Hz appear at 200 Hz in the decimated signal — you can't tell them apart! With anti-alias filtering, the 1800 Hz is removed before decimation, leaving only the clean 200 Hz component.

Key Takeaways

  • Downsampling by $M$: keep every $M$th sample; new rate = $f_s/M$; new Nyquist = $f_s/(2M)$.
  • Always apply anti-aliasing lowpass filter (cutoff ≤ $f_s/(2M)$) BEFORE downsampling.
  • In MATLAB: downsample(x, M) (no filter!) or decimate(x, M) (filter + downsample).
  • Aliasing from decimation is permanent — filtering afterwards cannot undo it.
