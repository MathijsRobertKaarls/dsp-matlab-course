6.5 — Zero Padding: Recap & Contrast with Zero Stuffing

Module 6 · Resampling · ~10 min

Introduction

This final topic consolidates the two "zero" operations that are frequently confused: zero padding and zero stuffing. Both involve adding zeros to a signal, but they operate in different dimensions, serve different purposes, and have completely different effects on the spectrum.

Side-by-Side Comparison

Property Zero Stuffing (6.4) Zero Padding (3.2)
Where zeros go Between existing samples At the end of the signal
Purpose Upsampling (rate conversion) Finer DFT frequency grid
Sampling rate Changes (×L) Unchanged
Frequency resolution Unchanged (same signal content) Unchanged (denser grid, not more info)
Spectral effect Creates periodic spectral images Interpolates spectrum (smoother curve)
Need post-filter? Yes — interpolation LPF to remove images No — directly used for FFT
MATLAB upsample(x, L) fft(x, M) with M > N
Memory trick: Zero stuffing stuffs the signal with zeros in the middle (like stuffing a sandwich). Zero padding pads the end (like padding the end of a form with blank space).

When Each is Used

Use Zero Stuffing when:

  • Converting from a lower to a higher sample rate.
  • First step in rational resampling (before the interpolation filter).
  • Implementing a polyphase filterbank.

Use Zero Padding when:

  • You want a smoother-looking spectrum for visualization.
  • You need to more accurately locate a spectral peak that falls between DFT bins.
  • You need to pad to the next power of 2 for FFT efficiency.
  • You need a certain DFT size for circular convolution via overlap-add/save methods.

Conceptual Questions

Q1. You want a smoother FFT plot from a 256-sample signal. You should:

  • Use upsample(x, 4) to zero-stuff before the FFT
  • Use fft(x, 1024) to zero-pad to 1024 points
  • Upsample the signal and recompute the FFT at the new rate
  • Record a longer signal — 256 samples is too short for any FFT

Q2. To upsample audio from 8 kHz to 16 kHz, you should:

  • Zero-pad the signal to twice its length and take the FFT
  • Zero-stuff by L=2 (upsample(x,2)) then apply a lowpass interpolation filter
  • Append the signal to itself
  • Apply a highpass filter at 8 kHz

Q3. What is the key fundamental difference between zero stuffing and zero padding?

  • Zero stuffing is in the frequency domain; zero padding is in the time domain
  • Zero stuffing requires a filter; zero padding never does
  • Zero stuffing changes the sample rate (upsampling); zero padding doesn't change the sample rate (only DFT grid density)
  • They are mathematically identical — just different terminology

MATLAB Exercise

%% 6.5 Zero Stuffing vs Zero Padding — Side-by-Side

fs = 200;  N = 32;
t  = (0:N-1)/fs;
f0 = 30;
x  = sin(2*pi*f0*t);

%% Method A — Zero Padding (FFT of long sequence)
M = 256;    % padded length
X_zp = fft(x, M);   % zero-padded FFT
f_zp = (0:M/2-1)*fs/M;

%% Method B — Zero Stuffing (upsample by L = M/N)
L = M/N;             % = 8
x_zs = upsample(x, L);    % zero stuff
b_i  = L * fir1(64, 1/L);
x_interp = filter(b_i, 1, x_zs);  % interpolation filter
fs_up = fs*L;
f_zs = (0:length(x_zs)/2-1)*fs_up/length(x_zs);

figure('Name','Zero Padding vs Zero Stuffing');

% Time domain
subplot(3,2,1); stem(t, x, 'filled', 'MarkerSize',4);
title(sprintf('Original: N=%d, fs=%d Hz', N, fs)); xlabel('Time (s)');

subplot(3,2,2); stem(t, x, 'filled', 'MarkerSize',4);
title('Same original'); xlabel('Time (s)');

% Operations
subplot(3,2,3);
x_pad = [x, zeros(1, M-N)];
stem(0:M-1, x_pad, 'filled', 'MarkerSize',2);
title(sprintf('Zero Padded to M=%d — zeros at end', M)); xlabel('Sample n');
xline(N-0.5, 'r--', 'zeros start');

subplot(3,2,4);
stem(0:L*N-1, x_zs, 'filled', 'MarkerSize',2);
title(sprintf('Zero Stuffed ×%d — zeros between samples', L)); xlabel('Sample n');

% Spectra
subplot(3,2,5);
plot(f_zp, abs(X_zp(1:M/2))*2/N);
xlabel('Hz'); ylabel('|X(f)|');
title('FFT of zero-padded signal (smooth spectrum, same rate fs)');
xline(f0,'r--','f0'); xlim([0 fs/2]);

subplot(3,2,6);
plot(f_zs, abs(fft(x_zs, length(x_zs)))*2/length(x_zs), 'DisplayName','Stuffed');
hold on;
plot(f_zs, abs(fft(x_interp, length(x_interp)))*2/length(x_interp), 'r', 'DisplayName','After interp filter');
xlabel('Hz'); ylabel('|X(f)|');
title(sprintf('FFT of zero-stuffed signal (rate = %d Hz): images visible', fs_up));
legend; xline(f0,'k--'); xlim([0 fs_up/2]);

% Conclusion
fprintf('\nZero Padding:  fs unchanged (%d Hz), DFT bin size = %.2f Hz\n', fs, fs/M);
fprintf('Zero Stuffing: fs increased to %d Hz, images at multiples of %d Hz\n', fs_up, fs/2);

Key Takeaways

  • Zero stuffing: zeros between samples → upsampling (rate ×L), creates spectral images, needs interpolation filter.
  • Zero padding: zeros at the end → finer DFT grid, rate unchanged, no filter needed.
  • Neither operation adds new signal information — they cannot improve true frequency resolution.
  • Confusing these two operations is a very common DSP mistake!
