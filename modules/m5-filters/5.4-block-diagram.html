5.4 — Block Diagrams

Module 5 · Filter Design · ~15 min

Introduction

A block diagram is the graphical language of DSP — a way to visualize signal flow through a system. Just as a circuit schematic describes an electronic system with standard symbols, a block diagram describes a DSP algorithm with standard building blocks. Reading and drawing block diagrams is an essential skill for communicating filter designs.

Reading a Block Diagram

Follow the signal flow arrows:

  1. Signals on arrows represent sequences (values at each sample $n$).
  2. Each $z^{-1}$ box introduces a one-sample delay.
  3. Triangles multiply by a constant coefficient.
  4. Circles sum their inputs.
  5. A line branching (fork) splits one signal into two paths — both carry the same value.

Signal Flow Graph (SFG)

An alternative notation uses nodes and directed edges (branches) with weights. Each node sums all incoming branch values. This compact representation is equivalent to the block diagram and is useful for formal analysis.

Mathematical Foundation

From a block diagram you can directly write the difference equation and the transfer function. For a second-order IIR (Direct Form I):

$$y[n] = b_0 x[n] + b_1 x[n-1] + b_2 x[n-2] - a_1 y[n-1] - a_2 y[n-2]$$ $$H(z) = \frac{b_0 + b_1 z^{-1} + b_2 z^{-2}}{1 + a_1 z^{-1} + a_2 z^{-2}}$$

Reading coefficients directly from the block diagram (multiplier weights) gives both representations simultaneously.

MATLAB Exercise

%% 5.4 Block Diagram → Difference Equation → Filter

%% Part A — Extract coefficients and draw pole-zero / freq response
% Design: 4th-order Butterworth lowpass at 200 Hz, fs=1000 Hz
[b, a] = butter(4, 200/(1000/2), 'low');

fprintf('=== 4th-order Butterworth Lowpass ===\n');
fprintf('b coefficients (feedforward): '); fprintf('%.6f ', b); fprintf('\n');
fprintf('a coefficients (feedback):    '); fprintf('%.6f ', a); fprintf('\n\n');

% Write out the difference equation
fprintf('Difference equation:\ny[n] = ');
for k = 0:length(b)-1
  if k == 0
    fprintf('%.4f*x[n]', b(k+1));
  else
    fprintf(' + %.4f*x[n-%d]', b(k+1), k);
  end
end
for k = 1:length(a)-1
  fprintf(' - %.4f*y[n-%d]', a(k+1), k);
end
fprintf('\n');

figure('Name','Block Diagram Verification');
subplot(1,2,1); zplane(b, a); title('Pole-Zero Plot');
subplot(1,2,2); freqz(b, a, 512, 1000); title('Frequency Response');

%% Part B — Second-order sections (SOS): numerically stable implementation
% For high-order filters, direct form can have numerical issues
% Split into cascaded second-order sections
[sos, g] = tf2sos(b, a);
fprintf('\nSecond-Order Sections (SOS):\n');
disp(sos);

% Apply using sosfilt (more numerically stable)
fs = 1000;  t = 0:1/fs:1-1/fs;
x = sin(2*pi*50*t) + sin(2*pi*400*t) + 0.3*randn(size(t));
y_direct = filter(b, a, x);
y_sos    = sosfilt(sos, x) * g;

fprintf('Max SOS vs Direct difference: %.2e\n', max(abs(y_direct - y_sos)));

figure('Name','Direct vs SOS');
subplot(2,1,1); plot(t(1:300), x(1:300)); title('Input: 50Hz + 400Hz + noise');
subplot(2,1,2);
plot(t(1:300), y_sos(1:300), 'b', 'DisplayName','SOS (stable)'); hold on;
plot(t(1:300), y_direct(1:300), 'r--', 'DisplayName','Direct form');
title('Filtered output'); legend;
💡 Hint

The difference equation printed in Part A translates directly to a block diagram: each b coefficient is a feedforward multiplier, each a coefficient is a feedback multiplier. The SOS form in Part B splits the high-order filter into cascaded 2nd-order "biquad" sections — this avoids numerical precision problems that can occur with high-order direct-form implementations.

Key Takeaways

  • Block diagrams use three symbols: delay ($z^{-1}$), multiplier, and adder.
  • Feedback loops → IIR; no feedback → FIR.
  • You can directly read off the difference equation and transfer function from a block diagram.
  • High-order filters should be implemented as cascaded second-order sections (SOS/biquad) for numerical stability.
  • In MATLAB: tf2sos(b,a) to convert, sosfilt(sos,x) to apply.
