5.1 — Filter Design & Analysis

Module 5 · Filter Design · ~25 min

Introduction

Filters are the workhorses of DSP. They selectively pass certain frequencies while attenuating others — removing noise, isolating a signal band, or shaping a spectrum. Understanding how to design, characterize, and choose filters is arguably the most practical skill in applied DSP.

Core Concept

Filter Types by Frequency Response

  • Lowpass: passes low frequencies, attenuates high frequencies. Used for noise removal, anti-aliasing.
  • Highpass: passes high frequencies, blocks low. Used for DC removal, edge detection.
  • Bandpass: passes a band, attenuates outside. Used for extracting one channel.
  • Bandstop (Notch): rejects a narrow band. Classic use: remove 50/60 Hz power line hum.

FIR vs IIR

Two fundamental filter architectures:

Property FIR (Finite Impulse Response) IIR (Infinite Impulse Response)
Feedback No (feedforward only) Yes (uses past outputs)
Stability Always stable Must check (poles inside unit circle)
Phase Linear phase possible Non-linear phase
Efficiency More coefficients needed Fewer coefficients for steep rolloff
Use case Audio, biomedical (phase-sensitive) Real-time control, audio EQ, comms

Filter Specifications

  • Passband: frequencies that should pass (gain ≈ 1 or 0 dB)
  • Stopband: frequencies to attenuate (gain ≪ 1, e.g. −60 dB)
  • Transition band: the roll-off region between pass and stopband
  • Ripple: variation in gain within the passband or stopband

Mathematical Foundation

FIR filter output (MA process):

$$y[n] = \sum_{k=0}^{M} b_k \, x[n-k]$$

IIR filter output (ARMA process):

$$y[n] = \sum_{k=0}^{M} b_k\, x[n-k] - \sum_{k=1}^{N} a_k\, y[n-k]$$

The frequency response (replace $z = e^{j\omega}$ in the Z-transform):

$$H(e^{j\omega}) = \frac{\sum_{k=0}^{M} b_k e^{-j\omega k}}{1 + \sum_{k=1}^{N} a_k e^{-j\omega k}}$$

Conceptual Questions

Q1. Why are FIR filters preferred for audio and biomedical applications?

  • They need fewer coefficients than IIR
  • They can achieve linear phase — all frequencies are delayed equally, preserving waveform shape
  • They have steeper roll-off for the same order
  • They are always faster to compute

Q2. To remove 50 Hz power line interference from an ECG recording, you would use a:

  • Lowpass filter with cutoff 50 Hz
  • Highpass filter with cutoff 50 Hz
  • Notch (bandstop) filter centered at 50 Hz
  • Bandpass filter passing 40–60 Hz

Q3. IIR filters can be unstable because they:

  • Have too many coefficients
  • Use feedback (past outputs), which can cause unbounded growth if poles are outside the unit circle
  • Always introduce nonlinear phase
  • Cannot be designed with standard methods

MATLAB Exercise

%% 5.1 Filter Design and Analysis

fs = 1000;   % Sampling frequency
Fc = 100;    % Cutoff frequency (Hz)
Fn = Fc/(fs/2);  % Normalized cutoff [0,1]

%% --- FIR filter (window method) ---
N_fir = 64;          % Filter order
b_fir = fir1(N_fir, Fn, 'low', hann(N_fir+1));  % Lowpass FIR

%% --- IIR Butterworth filter ---
[b_iir, a_iir] = butter(4, Fn, 'low');           % 4th-order Butterworth

%% --- Compare frequency responses ---
figure('Name','Filter Frequency Response');
[H_fir, f] = freqz(b_fir, 1, 1024, fs);
[H_iir, ~] = freqz(b_iir, a_iir, 1024, fs);

subplot(2,1,1);
plot(f, 20*log10(abs(H_fir)), 'b', 'DisplayName', sprintf('FIR order %d', N_fir)); hold on;
plot(f, 20*log10(abs(H_iir)), 'r', 'DisplayName', 'IIR Butterworth order 4');
xline(Fc, 'k--', 'fc=100Hz');
xlabel('Frequency (Hz)'); ylabel('Magnitude (dB)');
title('Lowpass: FIR vs IIR'); legend; ylim([-80 5]); grid on;

subplot(2,1,2);
plot(f, unwrap(angle(H_fir))*180/pi, 'b'); hold on;
plot(f, unwrap(angle(H_iir))*180/pi, 'r');
xlabel('Frequency (Hz)'); ylabel('Phase (degrees)');
title('Phase response — note FIR is linear, IIR is not'); legend('FIR','IIR'); xlim([0 fs/2]);

%% --- Apply filter to a noisy signal ---
t = 0:1/fs:2-1/fs;
x_clean = sin(2*pi*30*t);
x_noisy = x_clean + 0.6*randn(size(t)) + 0.4*sin(2*pi*300*t);

y_fir    = filter(b_fir, 1, x_noisy);
y_iir    = filter(b_iir, a_iir, x_noisy);
y_filtfilt = filtfilt(b_iir, a_iir, x_noisy);   % Zero-phase filtering

figure('Name','Filter Applied');
subplot(4,1,1); plot(t(1:500), x_noisy(1:500));  title('Noisy input'); ylim([-3 3]);
subplot(4,1,2); plot(t(1:500), y_fir(1:500));    title('FIR (linear phase — shifted)'); ylim([-2 2]);
subplot(4,1,3); plot(t(1:500), y_iir(1:500));    title('IIR filter() — causal, phase distortion'); ylim([-2 2]);
subplot(4,1,4); plot(t(1:500), y_filtfilt(1:500)); hold on;
plot(t(1:500), x_clean(1:500), 'r--');
title('IIR filtfilt() — zero-phase (offline only)'); ylim([-2 2]); legend('Filtered','True');

%% --- Design other filter types ---
% Highpass FIR
b_hp = fir1(64, 0.3, 'high', hann(65));
% Bandpass IIR
[b_bp, a_bp] = butter(3, [0.1 0.4], 'bandpass');
% Notch (50 Hz in 1000 Hz system)
Wo = 50/(fs/2);  BW = Wo/35;
[b_notch, a_notch] = iirnotch(Wo, BW);

figure('Name','Filter Types');
types = {b_fir,1; b_hp,1; b_bp,a_bp; b_notch,a_notch};
titles = {'Lowpass FIR','Highpass FIR','Bandpass IIR','Notch IIR (50Hz)'};
for i=1:4
  subplot(2,2,i);
  freqz(types{i,1}, types{i,2}, 1024, fs);
  title(titles{i});
end
💡 Hint

Note the key difference in the phase plot: FIR phase is a straight line (linear phase = constant group delay). IIR phase curves — different frequencies are delayed by different amounts, distorting the waveform shape. filtfilt() applies the filter twice (forward then backward) to achieve zero-phase — but this requires the complete signal (not real-time).

Key Takeaways

  • Filters classified by response: lowpass, highpass, bandpass, bandstop.
  • FIR: stable, linear phase possible, needs more coefficients. Design with fir1().
  • IIR: efficient, non-linear phase, feedback (can be unstable). Design with butter(), cheby1().
  • Apply with filter() (causal/real-time) or filtfilt() (zero-phase, offline).
  • Visualize with freqz(b, a, N, fs).
