5.3 — Difference Equation & Implementation

Module 5 · Filter Design · ~20 min

Introduction

A filter's behavior is most directly described by its difference equation — a recipe that tells you, sample by sample, how to compute each output from the inputs and previous outputs. This is what actually gets implemented in code or hardware. Understanding difference equations bridges the gap between abstract transfer functions and running code.

Core Concept

General Difference Equation

The general form of a causal LTI filter:

$$y[n] = \sum_{k=0}^{M} b_k\, x[n-k] \;-\; \sum_{k=1}^{N} a_k\, y[n-k]$$

Reading this as code: to compute the current output $y[n]$, take a weighted sum of the current and $M$ past inputs, then add/subtract $N$ weighted past outputs.

Special Cases

  • FIR (MA filter): $a_k = 0$ for all $k \geq 1$ — no feedback, pure feedforward.
  • IIR (AR filter): $b_k = 0$ for $k \geq 1$ — only current input, plus feedback.
  • ARMA: both $b_k$ and $a_k$ nonzero — most general IIR.

Direct Form I and II

Direct Form I: Implement the difference equation directly — two sets of delay lines (one for inputs, one for outputs). Simple to understand, but uses $M + N$ delay elements.

Direct Form II (transposed): Merges the two delay lines, requires only $\max(M, N)$ delays — more memory-efficient. This is what MATLAB's filter() uses internally.

Mathematical Foundation

Example: First-order lowpass IIR

The difference equation for a simple exponential smoothing filter:

$$y[n] = (1-\alpha)\, x[n] + \alpha\, y[n-1]$$

with $0 < \alpha < 1$. The corresponding transfer function:

$$H(z) = \frac{(1-\alpha)}{1 - \alpha z^{-1}}$$

Single pole at $z = \alpha$. As $\alpha \to 1$, the pole approaches the unit circle → narrower lowpass, slower response. As $\alpha \to 0$, the filter becomes a pass-through.

Implementing in a loop

% Direct implementation of: y[n] = b0*x[n] + b1*x[n-1] - a1*y[n-1]
function y = myfilter(b, a, x)
  N = length(x);
  y = zeros(1, N);
  for n = 1:N
    y(n) = b(1)*x(n);
    if n > 1; y(n) = y(n) + b(2)*x(n-1) - a(2)*y(n-1); end
    if n > 2; y(n) = y(n) + b(3)*x(n-2) - a(3)*y(n-2); end
  end
end

Conceptual Questions

Q1. An FIR filter's difference equation contains:

  • Only past output terms $y[n-k]$
  • Only input terms $x[n-k]$ — no feedback
  • Both input and feedback output terms
  • Only the current input $x[n]$

Q2. For the filter $y[n] = 0.9\,y[n-1] + x[n]$, computing $y[5]$ requires knowing:

  • Only $x[5]$
  • $x[5]$ and $x[4]$
  • $x[5]$ and $y[4]$ (which itself depends on all previous samples)
  • $x[5]$ through $x[0]$ directly

Q3. What is the advantage of Direct Form II over Direct Form I?

  • It gives a different (better) frequency response
  • It uses fewer delay elements — max(M,N) instead of M+N
  • It is always more numerically stable
  • It eliminates the feedback path

MATLAB Exercise

%% 5.3 Implementing Difference Equations

fs = 1000;
t  = 0:1/fs:1-1/fs;
x  = sin(2*pi*50*t) + 0.5*randn(size(t));   % signal + noise

%% Part A — Manual implementation vs filter()
% 2nd-order Butterworth lowpass at 100 Hz
[b, a] = butter(2, 100/(fs/2), 'low');
fprintf('b coefficients: '); fprintf('%.5f ', b); fprintf('\n');
fprintf('a coefficients: '); fprintf('%.5f ', a); fprintf('\n');

% Manual loop (Direct Form I)
N  = length(x);
y_manual = zeros(1, N);
for n = 1:N
  y_manual(n) = b(1)*x(n);
  if n >= 2; y_manual(n) = y_manual(n) + b(2)*x(n-1) - a(2)*y_manual(n-1); end
  if n >= 3; y_manual(n) = y_manual(n) + b(3)*x(n-2) - a(3)*y_manual(n-2); end
end

% MATLAB built-in filter()
y_builtin = filter(b, a, x);

fprintf('Max difference (manual vs filter): %.2e\n', max(abs(y_manual - y_builtin)));

figure('Name','Manual vs Built-in filter');
plot(t(1:300), y_manual(1:300), 'b', 'DisplayName','Manual loop');
hold on;
plot(t(1:300), y_builtin(1:300), 'r--', 'DisplayName', 'filter()');
xlabel('Time (s)'); title('Manual difference equation vs MATLAB filter()'); legend;

%% Part B — Exponential smoothing (1st order IIR)
alphas = [0.1, 0.5, 0.9, 0.99];
figure('Name','Exponential Smoothing');
for i = 1:4
  alpha = alphas(i);
  b_s = [1-alpha]; a_s = [1, -alpha];
  y_s = filter(b_s, a_s, x);
  subplot(2,2,i);
  plot(t(1:500), x(1:500), 'Color',[0.7 0.7 0.7]); hold on;
  plot(t(1:500), y_s(1:500), 'b', 'LineWidth', 1.5);
  title(sprintf('α = %.2f', alpha)); xlabel('Time (s)'); ylim([-3 3]);
end

%% Part C — Step response (reveals transient behaviour)
x_step = [zeros(1,50), ones(1,450)];  % step at n=50

figure('Name','Step and Impulse Response');
subplot(2,1,1);
y_step = filter(b, a, x_step);
stem(y_step(40:120), 'filled', 'MarkerSize', 3);
title('Step response of 2nd-order Butterworth');
xlabel('Sample'); xline(11,'r--','Step here');

subplot(2,1,2);
impz(b, a, 60);   % built-in impulse response
title('Impulse response h[n]');
💡 Hint

The max difference between manual and built-in should be machine precision (~1e-15) — they compute exactly the same thing. In Part B, small α (0.1) barely smooths; large α (0.99) produces very heavy smoothing with slow response. The step response in Part C shows overshoot because this is a 2nd-order system.

Key Takeaways

  • Difference equation: $y[n] = \sum b_k x[n-k] - \sum a_k y[n-k]$ — the implementation blueprint.
  • FIR: $a_k = 0$ (no feedback). IIR: nonzero $a_k$ (feedback from past outputs).
  • Direct Form I: straightforward, M+N delays. Direct Form II: efficient, max(M,N) delays.
  • In MATLAB: filter(b, a, x) for causal filtering; impz(b, a) for impulse response.
  • Manual loop implementation = same result as filter(), up to floating-point.
