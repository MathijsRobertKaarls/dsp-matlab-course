3.4 — Interpolation

Module 3 · Signal Quality · ~15 min

Introduction

Sometimes you need sample values between your existing samples — maybe to resample to a higher rate, to align two signals recorded at different rates, or to reconstruct a smooth waveform from coarse measurements. Interpolation estimates those in-between values from the samples you have.

The theoretically perfect method (sinc interpolation) comes from the Nyquist theorem itself. In practice, spline and linear methods offer good results with lower computation.

Core Concept

Common Interpolation Methods

  • Nearest-neighbor: use the value of the closest sample — fast but very rough (staircase).
  • Linear: straight line between adjacent samples — simple, smooth but introduces amplitude error in curved regions.
  • Spline (cubic): smooth polynomial through multiple points — much better curve-following.
  • Sinc (ideal): the theoretically perfect interpolation from the Nyquist theorem — requires infinite samples, approximated in practice.
The Nyquist theorem guarantees: A bandlimited signal can be perfectly reconstructed at any time instant from its samples using sinc interpolation. This is the mathematical foundation of the DAC.

Mathematical Foundation

Ideal (Sinc) Interpolation

The Nyquist reconstruction formula:

$$x(t) = \sum_{n=-\infty}^{\infty} x[n] \cdot \text{sinc}\!\left(\frac{t - nT_s}{T_s}\right)$$

where $\text{sinc}(u) = \sin(\pi u)/(\pi u)$. Each sample contributes a sinc "tent" centred at its sample time, and they sum to reconstruct the continuous signal exactly (assuming bandlimited signal, $f_{max} < f_s/2$).

Polynomial Interpolation

Linear: between samples $x[n]$ and $x[n+1]$, for $0 \leq \alpha < 1$:

$$x(nT_s + \alpha T_s) = (1-\alpha)\,x[n] + \alpha\,x[n+1]$$

Cubic spline: fits a piecewise cubic polynomial with continuous first and second derivatives at every sample — much smoother, no kinks.

Conceptual Questions

Q1. Which interpolation method is theoretically perfect for bandlimited signals?

  • Linear interpolation
  • Cubic spline
  • Sinc interpolation
  • Nearest-neighbor

Q2. Why is cubic spline generally better than linear interpolation for smooth signals?

  • Spline interpolation is always faster to compute
  • Spline uses smooth polynomials that respect curvature, reducing error in curved regions
  • Spline requires fewer samples
  • Spline avoids aliasing

Q3. Can interpolation reconstruct frequencies above the original Nyquist limit (fs/2)?

  • Yes, sinc interpolation can recover any frequency
  • Yes, if the polynomial order is high enough
  • No — interpolation cannot recover information that was never sampled
  • Yes, by zero-padding before interpolating

MATLAB Exercise

%% 3.4 Interpolation Methods

% Coarse samples
fs_low  = 100;    % Low sampling rate
fs_high = 1000;   % Target (fine) grid
T = 0.2;

t_low  = 0:1/fs_low:T;
t_high = 0:1/fs_high:T;

% True signal (bandlimited)
f0 = 30;
x_true = sin(2*pi*f0*t_high) + 0.5*sin(2*pi*2*f0*t_high);
x_low  = sin(2*pi*f0*t_low)  + 0.5*sin(2*pi*2*f0*t_low);

% Interpolation methods using interp1
methods = {'nearest', 'linear', 'spline', 'pchip'};
colors  = {'r', 'm', 'b', 'g'};

figure('Name','Interpolation Methods');
plot(t_high, x_true, 'k--', 'LineWidth', 2, 'DisplayName','True signal'); hold on;
plot(t_low, x_low, 'ko', 'MarkerSize', 6, 'DisplayName','Samples');

for i = 1:4
  xi = interp1(t_low, x_low, t_high, methods{i});
  err = rms(xi - x_true);
  plot(t_high, xi, colors{i}, 'DisplayName', sprintf('%s (RMS err=%.4f)', methods{i}, err));
end
xlabel('Time (s)'); ylabel('Amplitude'); legend; title('Interpolation comparison');

% Report errors
fprintf('\nInterpolation RMS errors:\n');
for i = 1:4
  xi = interp1(t_low, x_low, t_high, methods{i});
  fprintf('  %-10s : %.5f\n', methods{i}, rms(xi - x_true));
end

% Ideal sinc interpolation (MATLAB's resample = polyphase, close to sinc)
x_resamp = resample(x_low, fs_high, fs_low);
x_resamp = x_resamp(1:length(t_high));
fprintf('  %-10s : %.5f\n', 'resample', rms(x_resamp - x_true));
💡 Hint

Expected RMS error order (best to worst): resample (sinc-like) ≈ spline < linear < nearest. Nearest-neighbor will have the largest error because it ignores curvature entirely. Spline should come very close to the true signal for this smooth, bandlimited test signal.

Key Takeaways

  • Interpolation estimates signal values between samples.
  • Ideal sinc interpolation is perfect for bandlimited signals — approximated by resample() in MATLAB.
  • Cubic spline is a practical high-quality choice; linear is fast but less accurate.
  • Interpolation cannot recover information above the original Nyquist limit.
  • In MATLAB: interp1(t, x, t_new, 'spline') for general interpolation.
