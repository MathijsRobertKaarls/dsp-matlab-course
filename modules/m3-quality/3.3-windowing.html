3.3 — Taper Functions (Windowing)

Module 3 · Signal Quality · ~20 min

Introduction

When you take an FFT, you only have a finite chunk of a signal — as if you're looking through a rectangular window in time. The problem: abruptly cutting off a signal at both ends causes the spectrum to "smear" energy across many frequencies, obscuring weak peaks near strong ones. This is spectral leakage.

Windowing (tapering) gently fades the signal to zero at both edges, dramatically reducing leakage — at the cost of slightly worse frequency resolution. Choosing the right window is a fundamental engineering trade-off.

Core Concept

Why Leakage Happens

The rectangular window (no windowing) has a Fourier Transform shaped like a sinc function — with large sidelobes. Every frequency in your signal gets "smeared" by this sinc shape. A strong tone at 100 Hz contaminates neighboring frequencies at 101, 102 Hz etc.

Window Functions

Apply window $w[n]$ by pointwise multiplication before the FFT: $x_w[n] = x[n] \cdot w[n]$

Window Sidelobe level Main-lobe width Best for
Rectangular −13 dB Narrowest Transient detection (not for leakage)
Hann −31 dB Medium General purpose ← start here
Hamming −43 dB Medium Low-level signals near strong ones
Blackman −58 dB Widest Detecting weak signals near strong ones

Mathematical Foundation

Common window formulas

Hann window (often incorrectly called "Hanning"):

$$w_{Hann}[n] = 0.5\left(1 - \cos\!\frac{2\pi n}{N-1}\right), \quad 0 \leq n \leq N-1$$

Hamming window:

$$w_{Hamming}[n] = 0.54 - 0.46\cos\!\frac{2\pi n}{N-1}$$

Blackman window:

$$w_{Blackman}[n] = 0.42 - 0.5\cos\!\frac{2\pi n}{N-1} + 0.08\cos\!\frac{4\pi n}{N-1}$$

Effect on amplitude

Windowing scales down the signal (the window has mean < 1). To get correct amplitude estimates, normalize by the window's coherent gain (mean value):

$$|X_{corrected}| = \frac{|X_w[k]|}{\sum_n w[n] / N} \cdot \frac{2}{N}$$ In MATLAB: divide by mean(w) after windowing.

Conceptual Questions

Q1. Spectral leakage occurs because:

  • The sampling rate is too low
  • The finite rectangular window has a sinc-shaped frequency response with large sidelobes that smear energy across bins
  • Zero-padding introduces extra frequency components
  • The signal contains too many harmonics

Q2. You need to detect a very weak 200 Hz signal next to a strong 195 Hz signal. Which window is most appropriate?

  • Rectangular (narrowest main lobe)
  • Hann (general purpose)
  • Blackman (lowest sidelobes, best leakage rejection)
  • No window makes a difference here

Q3. When does the rectangular window produce zero spectral leakage?

  • Never — rectangular always causes leakage
  • When the signal contains an exact integer number of cycles within the analysis window
  • When N is a power of 2
  • When the signal is zero-padded before the FFT

MATLAB Exercise

%% 3.3 Window Functions and Spectral Leakage

fs = 1000;  N = 256;
t  = (0:N-1)/fs;

% Two tones: strong at 100 Hz, weak at 130 Hz (−40 dB)
f1 = 100; A1 = 1.0;
f2 = 130; A2 = 0.01;   % 40 dB weaker
x  = A1*sin(2*pi*f1*t) + A2*sin(2*pi*f2*t);

% Add fractional bin offset to guarantee leakage
f1 = 100.7;   % not on a bin
x  = A1*sin(2*pi*f1*t) + A2*sin(2*pi*f2*t);

windows = {'rectwin', 'hann', 'hamming', 'blackman'};
colors  = {'r','b','g','m'};
M = 2048;   % zero-pad for smooth display

figure('Name','Window Comparison');
for i = 1:4
  w  = window(str2func(windows{i}), N);
  xw = x .* w';
  Xw = fft(xw, M);
  f_ax = (0:M/2-1)*fs/M;
  plot(f_ax, 20*log10(abs(Xw(1:M/2))*2/sum(w)), colors{i}, 'DisplayName', windows{i});
  hold on;
end
xlabel('Frequency (Hz)'); ylabel('Magnitude (dBFS)');
title(sprintf('Spectral Leakage: strong at ~%.1fHz, weak at %dHz (−40dB)', f1, f2));
legend; xlim([60 200]); ylim([-80 5]);
yline(-40, 'k--', '-40 dB target'); grid on;

%% Window shapes in time domain
figure('Name','Window Shapes');
for i = 1:4
  w = window(str2func(windows{i}), N);
  plot(w, colors{i}, 'DisplayName', windows{i}); hold on;
end
xlabel('Sample n'); ylabel('w[n]'); title('Window Functions');
legend;

%% Frequency response of each window (sidelobes)
figure('Name','Window Frequency Response');
for i = 1:4
  w  = window(str2func(windows{i}), N);
  W  = fft(w, 4096);
  W_norm = abs(W) / max(abs(W));
  plot((0:2047)*fs/4096, 20*log10(W_norm(1:2048)), colors{i}, 'DisplayName', windows{i});
  hold on;
end
xlabel('Hz'); ylabel('dB'); title('Window Frequency Response (sidelobes)');
legend; xlim([0 150]); ylim([-80 5]); grid on;
💡 Hint

With the rectangular window, the 130 Hz weak tone is likely buried in the leakage from the 100.7 Hz tone. With Blackman, the sidelobes are low enough to reveal the −40 dB peak at 130 Hz. Compare the sidelobe levels in the third figure — Blackman's fall off much faster than rectangular's.

Key Takeaways

  • Spectral leakage: abrupt windowing smears energy into neighboring frequency bins.
  • Window functions taper the signal to zero at edges, reducing sidelobes.
  • Trade-off: lower sidelobes → wider main lobe → worse frequency resolution.
  • Hann: great general-purpose default. Blackman: best leakage rejection. Rectangular: best resolution when leakage isn't a concern.
  • In MATLAB: x .* hann(N)' or x .* hamming(N)'.
