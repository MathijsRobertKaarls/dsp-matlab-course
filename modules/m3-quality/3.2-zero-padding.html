3.2 — Zero-Padding

Module 3 · Signal Quality · ~15 min

Introduction

What if you could make your frequency plot look smoother and more detailed without collecting more data? That's what zero-padding offers — but there's an important catch. Zero-padding gives you a finer-looking spectrum, but it doesn't reveal frequencies that weren't there to begin with. Understanding this distinction prevents a very common mistake in DSP.

Core Concept

What is Zero-Padding?

Zero-padding means appending zeros to your signal before computing the DFT:

$$x_{ZP}[n] = \begin{cases} x[n] & 0 \leq n \leq N-1 \\ 0 & N \leq n \leq M-1 \end{cases}$$

where $M > N$ (usually $M = 2^k$ for FFT efficiency). The DFT of $x_{ZP}$ has $M$ bins instead of $N$, giving a finer frequency axis spacing of $\Delta f = f_s / M$.

The Key Distinction

⚠ Zero-padding ≠ better frequency resolution.
Frequency resolution (the ability to distinguish two closely spaced peaks) is determined by the signal duration: $\Delta f_{true} = f_s/N$. Zero-padding only provides interpolation between existing DFT bins — a smoother-looking spectrum, not a more informative one.

Think of it like zooming into a photo. Zooming in (zero-padding) makes the picture larger but doesn't add new detail if the original was low-resolution. To see more detail you need a higher-resolution photo (longer signal).

When is Zero-Padding Useful?

  • Peak location accuracy — if a frequency falls between DFT bins, zero-padding helps interpolate to find the true peak location more precisely.
  • Visualization — a smoother spectrum is easier to read and looks better in reports.
  • FFT efficiency — padding to the next power of 2 can speed up computation.
  • Circular convolution — padding prevents time-domain aliasing in block-wise convolution.

Mathematical Foundation

Zero-padding in the time domain corresponds to interpolation in the frequency domain. The $M$-point DFT of $x_{ZP}$ evaluates the DTFT at $M$ equally-spaced points instead of $N$:

$$X_{ZP}[k] = X(e^{j\hat\omega})\bigg|_{\hat\omega = 2\pi k/M}$$

Since DTFT is continuous and the $N$-point DFT evaluates it at $N$ points, zero-padding simply evaluates more points — it does not change the underlying DTFT shape.

True frequency resolution (Rayleigh criterion — minimum separation to distinguish two peaks):

$$\Delta f_{resolution} = \frac{f_s}{N} = \frac{1}{T_{record}}$$

Only collecting more samples ($N \uparrow$) or recording longer ($T_{record} \uparrow$) improves this.

Conceptual Questions

Q1. A signal is recorded for 0.1 s at 1000 Hz (N=100 samples). You zero-pad to N=1024. What is the true frequency resolution?

  • ~1 Hz (1000/1024)
  • 10 Hz (1000/100) — zero-padding doesn't improve true resolution
  • 0.1 Hz
  • 1000 Hz

Q2. Zero-padding is most useful for:

  • Resolving two closely spaced frequencies
  • More accurately locating the frequency of a single spectral peak between DFT bins
  • Reducing spectral leakage
  • Reducing noise in the time domain

Q3. Why does zero-padding NOT improve true frequency resolution?

  • Because zeros are filtered out by the FFT
  • Because the appended zeros contain no new signal information — we're only interpolating an existing DTFT
  • Because it changes the sampling frequency
  • Because it introduces aliasing

MATLAB Exercise

%% 3.2 Zero-Padding: Interpolation vs Resolution

fs = 1000;
N  = 64;       % Short signal — coarse true resolution
t  = (0:N-1)/fs;

% Two close frequencies: can we tell them apart?
f1 = 100;  f2 = 115;   % separation = 15 Hz; true resolution = 1000/64 ≈ 15.6 Hz
x = sin(2*pi*f1*t) + sin(2*pi*f2*t);

% Different padding lengths
pad_sizes = [N, 256, 1024, 4096];
colors = {'b','r','g','m'};

figure('Name','Zero-Padding Effect');
for i = 1:4
  M = pad_sizes(i);
  xp = [x, zeros(1, M-N)];      % zero-pad
  Xp = fft(xp);
  f_ax = (0:M/2-1)*fs/M;
  plot(f_ax, abs(Xp(1:M/2))*2/M, colors{i}, 'DisplayName', sprintf('M=%d', M));
  hold on;
end
xline(f1,'k--'); xline(f2,'k--');
xlabel('Frequency (Hz)'); ylabel('|X(f)|');
title(sprintf('N=%d samples, f1=%dHz, f2=%dHz (sep=%.0f Hz, resolution=%.1f Hz)', N, f1, f2, f2-f1, fs/N));
legend; xlim([60 160]);

% Now show that LONGER recording actually resolves the two peaks
figure('Name','True Resolution: Longer Signal');
for N2 = [32, 64, 128, 256]
  t2 = (0:N2-1)/fs;
  x2 = sin(2*pi*f1*t2) + sin(2*pi*f2*t2);
  X2 = fft(x2, 4096);    % same zero-padding for all, only signal length differs
  f2_ax = (0:2047)*fs/4096;
  plot(f2_ax, abs(X2(1:2048))*2/N2, 'DisplayName', sprintf('N=%d (Δf=%.1f Hz)', N2, fs/N2));
  hold on;
end
xline(f1,'k--'); xline(f2,'k--');
xlabel('Frequency (Hz)'); ylabel('|X(f)|');
title('Resolving two peaks: only longer recording helps (all zero-padded to 4096)');
legend; xlim([60 160]);
💡 Hint

In the first figure, all curves use the same N=64 samples with increasing zero-padding. The curves get smoother, but the two peaks only become distinguishable when N is large enough (true resolution ≤ frequency separation). In the second figure, even with the same zero-padding, a longer signal (larger N) reveals two distinct peaks.

Key Takeaways

  • Zero-padding appends zeros, increasing the DFT size from $N$ to $M > N$.
  • Result: smoother, interpolated spectrum — useful for peak localization and visualization.
  • True frequency resolution $\Delta f = f_s/N$ depends only on signal length, NOT padding.
  • To distinguish two close frequencies, you need more signal samples, not more zeros.
  • In MATLAB: fft(x, M) automatically zero-pads to length M.
