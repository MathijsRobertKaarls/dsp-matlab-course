7.4 — MATLAB Lifehacks & Tips

Module 7 · Reference & Summary · ~20 min

Introduction

The course taught the DSP. This page collects the practical MATLAB craft that makes the DSP painless: idioms, gotchas, and habits that save hours. These are the things experienced users do automatically — copy them into your muscle memory.

1 · Vectorize — Avoid Loops

MATLAB is fastest when you operate on whole arrays. A loop that builds a sine sample-by-sample is both slower and harder to read than the vectorized form.

% Slow, un-MATLAB-like
for n = 1:N
  x(n) = sin(2*pi*f*(n-1)/fs);
end

% Fast, idiomatic — element-wise operators (.* ./ .^)
t = (0:N-1)/fs;        % row vector of times
x = sin(2*pi*f*t);     % whole signal at once
The dot matters: * / ^ are matrix operations; .* ./ .^ are element-wise. For signals you almost always want the dotted versions. a.*b multiplies sample-by-sample; a*b tries a matrix product and usually errors.

2 · Preallocate When You Must Loop

If a loop is unavoidable (e.g. a real-time filter, an iterative algorithm), preallocate the output so MATLAB doesn't resize the array every iteration.

y = zeros(1, N);          % preallocate — not y = []
for n = 1:N
  y(n) = someRecursion(n);
end

3 · Build Frequency & Time Axes Correctly

The single most common beginner bug is a mislabelled axis. Memorize these two lines:

t = (0:N-1)/fs;            % time axis (seconds), length N
f = (0:N-1)*(fs/N);       % full DFT frequency axis (Hz), length N
% one-sided axis for a real signal:
f1 = (0:floor(N/2)) * fs/N;
Why (0:N-1) and not (1:N)? DSP is 0-indexed in theory ($x[0]$ is the first sample) but MATLAB arrays are 1-indexed (x(1) is the first sample). Keep the math 0-based for axes, but index arrays from 1. Off-by-one here shifts your whole frequency axis.

4 · Normalized Frequencies in Filter Design

Filter functions take cutoff frequencies normalized to Nyquist ($f_s/2$), not to $f_s$. This trips up nearly everyone.

fs = 1000;  Fc = 100;        % want 100 Hz cutoff
Wn = Fc / (fs/2);            % = 0.2  (normalized to Nyquist)
[b,a] = butter(4, Wn, 'low');

For a bandpass: Wn = [f_lo f_hi]/(fs/2). A value of 1.0 means Nyquist; values must be strictly between 0 and 1.

5 · Reproducible Randomness

Seed the generator so your noisy experiments are repeatable — essential when comparing two methods.

rng(42);                  % fixed seed → same randn() every run
x = signal + 0.5*randn(size(signal));

6 · freqz Without Arguments Hijacks Your Figure

Calling freqz(b,a) with no output arguments builds its own two-panel figure and will clobber any subplot grid you set up. To plot inside your own layout, capture the outputs:

[H,f] = freqz(b, a, 1024, fs);   % capture, don't auto-plot
subplot(2,1,1);
plot(f, 20*log10(abs(H)));       % your own magnitude panel
grid on; xlabel('Hz'); ylabel('dB');

7 · Two-Sided vs One-Sided Spectra

The FFT returns a two-sided spectrum (0 → fs). For a real signal the upper half mirrors the lower, so a one-sided plot doubles the interior bins to conserve energy:

X  = fft(x);
P2 = abs(X)/N;
P1 = P2(1:floor(N/2)+1);
P1(2:end-1) = 2*P1(2:end-1);    % double all but DC and Nyquist
f  = (0:floor(N/2))*fs/N;
plot(f, P1);
Don't double DC or Nyquist: bin 0 (DC) and bin N/2 (Nyquist, when N is even) have no mirror partner — doubling them inflates their amplitude. That's why the slice is 2:end-1.

8 · Power-of-Two FFT Lengths

FFTs are fastest when the length is a power of 2. Pad up with nextpow2:

M = 2^nextpow2(N);     % next power of two ≥ N
X = fft(x, M);         % zero-pads to M automatically

10 · Row vs Column Vectors

Since R2016b, MATLAB broadcasts mismatched shapes silently — a row times a column gives a matrix, not an error. This produces gigantic arrays and wrong results.

r = 1:5;        % 1×5 row
c = (1:5)';     % 5×1 column
r .* c          % ⚠ 5×5 matrix via broadcasting — probably NOT what you want!
r .* r          % ✓ 1×5, correct

Habit: keep signals as consistent row (or consistent column) vectors. Use x(:)' to force a row.

11 · Plotting Niceties

grid on;                       % always — easier to read values
xlabel('Frequency (Hz)'); ylabel('Magnitude (dB)');
title('My spectrum');
legend('FIR','IIR','Location','best');
xlim([0 fs/2]); ylim([-80 5]);
yline(-3,'k--','-3 dB');       % reference lines
xline(Fc,'r--','f_c');
hold on;  % overlay multiple curves; remember hold off afterwards
dB without −Inf warnings: 20*log10(abs(H)) hits -Inf where H is exactly 0 (e.g. an FIR null). Add a tiny floor: 20*log10(abs(H)+eps).

Key Takeaways

  • Vectorize with .*/.//.^; preallocate when you must loop.
  • Normalize filter cutoffs to Nyquist (Fc/(fs/2)); build axes from (0:N-1).
  • Capture freqz outputs to control your own plots; double only interior bins for one-sided spectra.
  • Watch row-vs-column broadcasting; seed rng for reproducibility; check toolboxes with ver.
