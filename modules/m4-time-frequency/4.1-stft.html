4.1 — STFT & Spectrogram

Module 4 · Time-Frequency Analysis · ~20 min

Introduction

The standard FFT gives you a single spectrum for the entire signal — it tells you what frequencies are present but not when. For a sustained tone, that's fine. But for speech, music, or a chirp (a frequency-sweeping signal), you need to know how the spectrum changes over time.

The Short-Time Fourier Transform (STFT) solves this by sliding a window along the signal and computing the FFT at each position. The result is a 2D grid of frequency vs. time — the spectrogram.

Core Concept

How the STFT Works

  1. Divide signal into short, overlapping segments using a window function.
  2. Compute the FFT of each windowed segment.
  3. Stack the spectra as columns → 2D time-frequency representation.

The Time-Frequency Trade-Off

The STFT reveals a fundamental tension in time-frequency analysis:

Heisenberg-Gabor Uncertainty: You cannot simultaneously have perfect time resolution and perfect frequency resolution. A shorter window gives better time resolution but smears frequency. A longer window gives better frequency resolution but loses time detail.
  • Short window: good at locating events in time, poor frequency detail.
  • Long window: good frequency detail, poor time localization.

This trade-off is fixed for the STFT — the window shape and length are chosen once. Wavelets (Topic 4.2) address this limitation.

Mathematical Foundation

The STFT of a discrete signal $x[n]$ with window $w[n]$:

$$X_{STFT}[m, k] = \sum_{n=-\infty}^{\infty} x[n]\, w[n - mH]\, e^{-j2\pi kn/N}$$

where:

  • $m$ — time frame index (which window position)
  • $H$ — hop size (samples between successive window positions)
  • $N$ — window length (FFT size)
  • $k$ — frequency bin index

Overlap: $\text{overlap\%} = (1 - H/N) \times 100\%$. Common choices: 50% or 75%.

Spectrogram = $|X_{STFT}[m, k]|^2$ (power) or $|X_{STFT}[m, k]|$ (magnitude), displayed as a heatmap.

Conceptual Questions

Q1. What is the main advantage of the STFT over a single FFT?

  • It computes faster than a regular FFT
  • It shows how the frequency content of a signal changes over time
  • It eliminates the need for windowing
  • It has better frequency resolution than the FFT

Q2. Increasing the STFT window length has what effect?

  • Better time resolution, worse frequency resolution
  • Better frequency resolution, worse time resolution
  • Better both time and frequency resolution
  • No effect — the trade-off is controlled by hop size

Q3. How does a linearly increasing chirp (frequency sweeping from 100 to 500 Hz) appear on a spectrogram?

  • A single vertical line at 300 Hz (the average frequency)
  • A flat horizontal band across all time
  • A diagonal line rising from low to high frequency as time progresses
  • Multiple vertical stripes at 100, 200, 300, 400, 500 Hz

MATLAB Exercise

%% 4.1 STFT and Spectrogram

fs = 4000;
T  = 2;
t  = 0:1/fs:T-1/fs;

%% Part A — Chirp signal (sweeping frequency)
f_start = 100;  f_end = 1500;
x_chirp = chirp(t, f_start, T, f_end);

% Spectrogram with different window lengths
figure('Name','Spectrogram — Window Length Effect');
win_sizes = [64, 256, 1024];
for i = 1:3
  subplot(1,3,i);
  spectrogram(x_chirp, hann(win_sizes(i)), round(win_sizes(i)*0.75), win_sizes(i)*4, fs, 'yaxis');
  title(sprintf('Window N=%d', win_sizes(i)));
  colormap('hot'); colorbar off;
end

%% Part B — Multi-component non-stationary signal
x_talk = sin(2*pi*300*t(1:fs))   ;   % 300 Hz for first second
x_talk = [x_talk, sin(2*pi*800*t(1:fs))];   % 800 Hz for second second
t2 = (0:length(x_talk)-1)/fs;

figure('Name','Time-Varying Signal Spectrogram');
subplot(2,1,1);
plot(t2, x_talk);
xlabel('Time (s)'); ylabel('Amplitude');
title('Signal: 300 Hz (first second) → 800 Hz (second second)');

subplot(2,1,2);
spectrogram(x_talk, hann(512), 384, 2048, fs, 'yaxis');
title('Spectrogram (reveals the transition)');
colormap('jet');

%% Part C — Manual STFT (understand the internals)
N_win = 256;  hop = 64;
x_test = x_chirp;
n_frames = floor((length(x_test) - N_win) / hop) + 1;
STFT = zeros(N_win/2, n_frames);
w = hann(N_win);

for m = 1:n_frames
  idx = (m-1)*hop + (1:N_win);
  frame = x_test(idx) .* w';
  X = fft(frame, N_win);
  STFT(:, m) = abs(X(1:N_win/2));
end

t_ax = (0:n_frames-1)*hop/fs;
f_ax = (0:N_win/2-1)*fs/N_win;

figure('Name','Manual STFT');
imagesc(t_ax, f_ax, 20*log10(STFT + 1e-6));
axis xy; colormap('hot');
xlabel('Time (s)'); ylabel('Frequency (Hz)');
title('Manual STFT of chirp'); colorbar;
💡 Hint

In Part A, compare the three window sizes. N=64: poor frequency detail (thick chirp line), good time resolution. N=1024: thin, precise chirp line in frequency but smeared in time. N=256 is usually a good compromise. Part B should clearly show the jump from one horizontal line to another at t=1s.

Key Takeaways

  • The STFT computes short-time FFTs along a sliding window, producing a 2D time-frequency representation.
  • The spectrogram ($|STFT|^2$) shows how frequency content evolves over time.
  • Fundamental trade-off: short window = good time res, poor freq res; long window = vice versa.
  • In MATLAB: spectrogram(x, window, overlap, nfft, fs).
  • Hop size and overlap control time axis density; window length controls frequency resolution.
