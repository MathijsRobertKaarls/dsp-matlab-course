1.4 — Causal, Linear & Time-Invariant Systems

Module 1 · Foundations · ~25 min

Introduction

A system takes a signal in and produces a signal out. Your smartphone's equalizer is a system. A filter that removes power-line hum is a system. Even the act of averaging consecutive samples is a system.

To analyze and design systems rigorously, we classify them by their properties. The most important class in DSP is the LTI system — Linear and Time-Invariant. Understanding why this class is special unlocks the most powerful tools in DSP: the Fourier Transform, the Z-Transform, and convolution.

Core Concept

What is a System?

A system $\mathcal{H}$ maps an input signal $x[n]$ to an output signal $y[n]$:

$$y[n] = \mathcal{H}\{x[n]\}$$

Linearity

A system is linear if it satisfies two properties:

  1. Superposition (additivity): If input $x_1$ produces $y_1$ and input $x_2$ produces $y_2$, then input $x_1 + x_2$ produces $y_1 + y_2$.
  2. Homogeneity (scaling): If input $x$ produces output $y$, then input $ax$ produces output $ay$ for any constant $a$.

Together: $\mathcal{H}\{a \cdot x_1[n] + b \cdot x_2[n]\} = a \cdot y_1[n] + b \cdot y_2[n]$

Analogy: A linear amplifier that doubles voltage: input 3V → 6V, input 5V → 10V, input 3+5=8V → 16V. Perfectly linear. A clipping amplifier that saturates at 8V is not linear (8V+anything → 8V).

Time-Invariance

A system is time-invariant if a delayed input produces the same output, just delayed by the same amount:

$$\text{If } x[n] \to y[n], \text{ then } x[n - k] \to y[n - k] \text{ for any integer } k$$

In other words: the system doesn't care when you apply the input — the relationship between input and output stays the same at all times. A thermostat set to 20°C is time-invariant. A timer that behaves differently in the morning than at night is not.

Causality

A system is causal if the output at time $n$ depends only on present and past inputs, never on future inputs:

$$y[n] = f(x[n], x[n-1], x[n-2], \ldots)$$

This is a physical necessity for real-time systems — you can't know the future! However, in offline processing (post-recording), non-causal filters are allowed and often preferred (they can have zero phase distortion).

Example — Non-causal filter: A centered moving average: $y[n] = \frac{1}{3}(x[n-1] + x[n] + x[n+1])$ uses a future sample $x[n+1]$. This is fine for processing a recorded ECG but impossible for live audio.

Why LTI is So Powerful: Convolution

For an LTI system, the output is always the convolution of the input with the system's impulse response $h[n]$:

$$y[n] = x[n] * h[n] = \sum_{k=-\infty}^{\infty} x[k] \cdot h[n-k]$$

The impulse response $h[n]$ completely characterizes an LTI system. Feed in one sharp impulse, measure the output — and you know everything about how that system will respond to any input.

BIBO Stability

A system is BIBO stable (Bounded Input → Bounded Output) if every bounded input produces a bounded output. For LTI systems, this requires:

$$\sum_{n=-\infty}^{\infty} |h[n]| < \infty$$

Unstable systems amplify inputs without bound — catastrophic in practice (think audio feedback squeal, or structural resonance in bridges).

Mathematical Foundation

The Unit Impulse

$$\delta[n] = \begin{cases} 1 & n = 0 \\ 0 & n \neq 0 \end{cases}$$

Any signal can be expressed as a weighted sum of shifted impulses:

$$x[n] = \sum_{k=-\infty}^{\infty} x[k] \cdot \delta[n - k]$$

Convolution Sum

Because the system is linear, it processes each impulse component independently; because it's time-invariant, a shifted impulse produces a shifted response:

$$y[n] = \sum_{k=-\infty}^{\infty} x[k] \cdot h[n-k] \triangleq (x * h)[n]$$

Key property: Frequency domain

In the frequency domain, convolution becomes multiplication:

$$Y(e^{j\omega}) = X(e^{j\omega}) \cdot H(e^{j\omega})$$

This is why filters are easy to design in the frequency domain — just multiply your signal's spectrum by the filter's frequency response!

Conceptual Questions

Q1. Which system is not linear?

  • $y[n] = 3 \cdot x[n]$ (scaling)
  • $y[n] = x[n] + x[n-1]$ (moving average)
  • $y[n] = x[n]^2$ (squaring)
  • $y[n] = x[n] - x[n-2]$ (differencing)

Q2. Which system is non-causal?

  • $y[n] = x[n]$
  • $y[n] = x[n] - x[n-1]$
  • $y[n] = x[n+1]$
  • $y[n] = 0.9 \cdot y[n-1] + x[n]$

Q3. Why is the impulse response so important for LTI systems?

  • It describes the system's output only for sine wave inputs
  • It completely characterizes the system — any output can be computed by convolving the input with it
  • It is required only for causal systems
  • It describes how fast the system responds to changes in sampling rate

MATLAB Exercise

Compute and visualize convolution. Design a simple moving average filter and observe its impulse response.

%% 1.4 Convolution and Impulse Response

%% Part A — Manual convolution: moving average filter
% Impulse response: equal-weight average of M samples
M = 5;
h = ones(1, M) / M;   % FIR low-pass filter (moving average)

% Generate a noisy signal
fs = 1000; t = 0:1/fs:1-1/fs;
x_clean = sin(2*pi*50*t);                    % 50 Hz sine
x_noisy = x_clean + 0.8*randn(size(t));      % + noise

% Apply filter via convolution
y = conv(x_noisy, h, 'same');   % 'same' keeps output same length as input

figure('Name','Moving Average Filter');
subplot(3,1,1); plot(t, x_clean); title('Clean signal'); ylim([-3 3]);
subplot(3,1,2); plot(t, x_noisy); title('Noisy signal'); ylim([-3 3]);
subplot(3,1,3); plot(t, y);       title(sprintf('After moving average (M=%d)', M)); ylim([-3 3]);
xlabel('Time (s)');

%% Part B — Impulse response
delta = [1 zeros(1, 49)];   % Unit impulse: 1 at n=0, zeros elsewhere
h_out = conv(delta, h, 'full');

figure('Name','Impulse Response');
stem(h_out(1:10), 'filled');
title('Impulse response of moving average filter');
xlabel('Sample n'); ylabel('h[n]');

%% Part C — Test linearity manually
a = 2; b = -1;
x1 = sin(2*pi*100*t);
x2 = sin(2*pi*200*t);
y_combined   = conv(a*x1 + b*x2, h, 'same');
y_superposed = a*conv(x1, h, 'same') + b*conv(x2, h, 'same');

fprintf('Max difference (should be ~0): %.2e\n', max(abs(y_combined - y_superposed)));
💡 Hint

In Part A, the moving average smooths the noisy signal — but it also blurs sharp features. Larger $M$ = more smoothing but more lag. In Part B, the impulse response of an M-point moving average is simply M equal-height values of $1/M$ — a rectangular shape. In Part C, the max difference should be essentially 0 (floating point noise only), confirming linearity.

Key Takeaways

  • A system maps input $x[n]$ to output $y[n]$; described by its properties: linear, time-invariant, causal.
  • Linearity: superposition holds — $\mathcal{H}\{ax_1 + bx_2\} = a\mathcal{H}\{x_1\} + b\mathcal{H}\{x_2\}$.
  • Time-invariance: shift input → same output, shifted.
  • Causality: output depends only on present and past inputs.
  • LTI systems are fully described by their impulse response $h[n]$; output = convolution $y = x * h$.
